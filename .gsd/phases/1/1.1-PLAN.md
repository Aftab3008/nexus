# Plan 1.1: Backend Schema & Mutations

## Objective

Implement the backend foundation for the Letters feature, including database schema and API functions.

## Context

- .gsd/SPEC.md
- .gsd/DECISIONS.md
- convex/schema.ts
- convex/letters.ts

## Tasks

<task type="auto">
  <name>Update Schema</name>
  <files>convex/schema.ts</files>
  <context>
    - Use `mcp_convex` tools (`envList`, `tables`, `functionSpec`) to understand existing setup.
    - Use `mcp_clerk` tools (`clerk_sdk_snippet`) if needing to verify user identity fields.
    - Use `code-simplifier` skill to ensure clean schema definition.
  </context>
  <action>
    - Define `letters` table
    - Fields:
      - `senderId` (v.id("users"))
      - `receiverId` (v.id("users"))
      - `coupleId` (v.id("couples"))
      - `body` (v.string()) - stores rich text JSON/HTML
      - `type` (v.union(v.literal("text"), v.literal("image")))
      - `isScheduled` (v.boolean())
      - `scheduledAt` (v.optional(v.number()))
      - `deliveredAt` (v.optional(v.number()))
      - `isRead` (v.boolean())
      - `status` (pending, delivered)
    - Indexes: `by_couple`, `by_sender_status`, `by_receiver_status`
  </action>
</task>
